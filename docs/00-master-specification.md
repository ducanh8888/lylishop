# LyliShop Website Revamp - Master Specification

Version: 1.0  
Status: Source of truth for all next tasks  
Current website: https://lylishop.online  
Reference website: https://lenlyhandmade.shop

## 1. Project Goal

The goal is not to copy the reference website.

The reference website may only be used to learn:

- Interface layout logic.
- Section order.
- User experience.
- Navigation.
- Conversion journey.
- Landing Page structure.
- Image distribution.
- Content presentation.

Do not copy:

- Logo.
- Content.
- Images.
- Font.
- Colors.
- Brand identity.

## 2. Website Type

LyliShop must remain the same type of website.

LyliShop is a Landing Page.

Do not:

- Convert it into an ecommerce website.
- Change the website model.
- Change the project architecture.
- Change Next.js App Router.
- Change routing.
- Change existing data.

## 3. Design Goal

The completed website should feel similar to the reference website only in terms of:

- Section order.
- Spacing.
- Section division.
- Navigation.
- User journey.
- CTA placement.
- Content presentation.

It must not be similar in terms of:

- Logo.
- Images.
- Content.
- Colors.
- Font.
- Branding.

## 4. Brand Identity

Use the existing LyliShop SVG logo as the brand source.

All colors must come from the logo.

Do not:

- Use colors from the reference website.
- Copy the reference brand identity.
- Invent a new brand palette without approval.

## 5. Main Menu

The main website menu must contain only:

- Trang chủ
- Tin tức
- Cửa hàng
- Giới thiệu
- Liên hệ

Do not add unnecessary menu items.

## 6. Conversion Flow

LyliShop does not sell directly on the website.

Do not create:

- Checkout.
- Payment.
- Login.
- Register.
- Wishlist.
- Cart.

The website flow is:

1. Introduce products.
2. Build trust.
3. User clicks contact CTA.
4. User is sent to Facebook, Zalo, or Instagram.
5. Order is closed outside the website.

## 7. Homepage Structure

The homepage must display sections in this exact order:

1. Header.
2. Hero Banner.
3. Danh mục sản phẩm.
4. Sản phẩm bán chạy.
5. Đánh giá khách hàng.
6. Tin tức.
7. Footer.

## 8. Product Categories

The displayed categories must include:

- Móc khóa Mini.
- Móc khóa Size S.
- Móc khóa Size M.
- Móc khóa Size L.
- Thú bông len.
- Hoa len.

## 9. News

The homepage should show only 2 featured articles.

Suggested article titles:

1. Móc khóa len handmade - Món quà nhỏ mang ý nghĩa lớn
2. 5 lý do móc khóa len handmade được nhiều người lựa chọn làm quà tặng

Do not copy articles from the reference website.

## 10. Footer

The footer must include:

- Thông tin liên hệ.
- Chính sách đổi trả.
- Chính sách vận chuyển.
- Điều khoản.
- Fanpage Facebook.
- Instagram.
- Zalo.
- Số điện thoại.
- Email.
- Google Maps.

## 11. About Section

The Giới thiệu area must include 3 blocks displayed in one row on desktop:

- Sứ mệnh.
- Giá trị cốt lõi.
- Định vị thương hiệu.

## 12. Contact Section

The contact area must include:

- Contact form.
- Facebook.
- Instagram.
- Zalo.
- Email.
- Phone.
- Google Maps.

Do not create order functionality on the website.

## 13. Code Rules

Do not:

- Change project structure without a clear reason.
- Change routing.
- Change framework.
- Change existing data.
- Delete components without a clear reason.

Prefer:

- Reusing existing components.
- Keeping changes tightly scoped to the current task.
- Preserving working features.

## 14. UI Rules

The reference website may be studied for:

- Layout.
- Spacing.
- Section structure.
- Grid.
- Responsive behavior.
- Animation behavior.
- Card patterns.
- Button placement.

The implementation must still use LyliShop's own visual style.

## 15. SEO Rules

Do not reduce current SEO quality.

Every change must consider:

- Title.
- Description.
- Heading.
- Internal link.
- Image alt.
- Schema.
- JSON-LD.
- Canonical.
- OpenGraph.
- Breadcrumb.
- Core Web Vitals.

## 16. Task Output Rules

Every task must include:

- Files changed.
- Goal.
- Scope confirmation.
- Confirmation that unrelated files were not changed.

A task is complete only when all requirements of that task are satisfied.

## 17. Absolute Rule For Future Work

All future tasks must follow this specification unless the owner explicitly approves an update to this document.
